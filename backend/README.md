# DevStore Backend: High-Performance Logic Layer

The logic layer of DevStore is a FastAPI-powered application designed for low-latency resource discovery and AI-assisted evaluation. It serves as the bridge between the frontend and the serverless data stack (Neon & Pinecone).

---

## 🏗️ Architecture & Stack

- **Framework:** FastAPI (Python 3.11+)
- **Serverless PostgreSQL:** Neon (Managed Metadata & Structured Search)
- **Vector Database:** Pinecone (Semantic Retrieval)
- **AI/LLM:** AWS Bedrock (Claude 3.5 Sonnet & Nova Micro)
- **Inference:** Amazon Titan Embeddings

## 🚀 Setup & Installation

### Prerequisites
- Python 3.11+
- Pinecone API Key & Index
- Neon Database URI
- AWS Credentials (with Bedrock access)

### Local Development
1. **Initialize Virtual Environment:**
   ```bash
   python -m venv venv
   source venv/bin/activate  # Windows: venv\Scripts\activate
   ```

2. **Install Dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Configure Environment:**
   ```bash
   cp .env.example .env
   # Populate .env with PINECONE_API_KEY, NEON_DATABASE_URL, and AWS_REGION
   ```

4. **Launch Server:**
   ```bash
   uvicorn main:app --reload --port 8000
   ```

## 🛠️ Key Components

- **`main.py`**: The application entry point (Mangum-wrapped for Lambda compatibility).
- **`services/search.py`**: Implementation of the **Dual-Layer Search** (SQL + Vector).
- **`rag/`**: The core RAG engine handling context injection and Bedrock orchestration.
- **`ingestion/`**: Unified data harvesters for GitHub, HuggingFace, and Kaggle.
- **`clients/`**: Native SDK wrappers for Bedrock, Neon, and Pinecone.

## 📊 Dual-Layer Retrieval Logic

The backend follows a strict high-performance search flow:
1. **SQL Filter:** Neon PostgreSQL executes a metadata-aware query for category, pricing, and platform filters.
2. **Vector Rank:** Pinecone performs a K-Nearest Neighbor (KNN) search using query embeddings generated by Amazon Titan.
3. **Composite Scoring:** A ranking service normalizes results based on semantic relevance, popularity (Stars/Downloads), and freshness.
